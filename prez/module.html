Java Module
Module descriptor
module-info.java :
module foo {
  requires java.base;
  requires bar;
}
requires ==  module dependencies
exports == packages visible
                          by others modules 
module foo {
  requires java.base;
  requires bar;

  exports com.acme.my.packages
}
Exports
Open Package
Deep reflection must be allowed explicitly
module foo.bean {
  exports foo.bean.business;
   opens foo.bean.business;
}
Open Module
To open all packages* of a module
* even the non visible ones
open module foo.bean {
  exports foo.bean.business;
}
Layout on disc
legacy layout:
    1 project -> 1 module
    javac --source-path ...

multi-module layout:
    1 project -> n modules
    javac --module-source-path ...
--module-source-path
javac -d target/main/exploded \
         --module-source-path src/main/java \
         $(find src/main/java -name "*.java")
src/main/java
   my.first.module
      module-info.java
      com.foo.package
          Main.java
   another.module
      module-info.java
      ...
Module artifact
jar --create \
     --file target/main/artifact/my.first.module-1.0.jar \
     --module-version=1.0 \
     --main-class com.foo.package.Main \
     -C target/exploded/my.first.module .
A modular jar contains a module-info.class
ClassPath vs ModulePath
An unamed module contains all jars
   from the class-path
Modules are found in the module-path
a module can only see its required modules
an unamed module see all other modules
Automatic module
It's a non modular jar in the module-path
it can an be required by another module
  the name is the name of the jar*
it exports all its packages
it requires all other modules + classpath
     automatically
Service
Provides an implementation of an interface
module bar {
  requires foo.bean;

  provides foo.bean.AnInterface
     with bar.AnImplementation;
}
In module baz, the java.util.ServiceLoader
will find the implementation at runtime
Uses
Find all implementations of an interface
module baz {
  uses foo.bean.AnInterface;
}
